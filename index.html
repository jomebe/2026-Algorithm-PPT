1. 개요

기획 의도

목표: 학교 급식실 도달하기

- 친숙한 학교 복도를 모티브로 설계한 3D 레이스 게임.
- 임의로 주행을 방해하며 움직이는 학생 NPC들을 피해 목적지에 무사 도달하는 것이 골자.

1. 개요

메인 테마

확률 기반 NPC 이동

불규칙한 확률 이동으로 예측 불가능한 주행 코스를 만듭니다.

익숙한 학교 맵

실제 학교 건물 통로 구조를 본떠 맵 공간을 조형했습니다.

1. 개요

주제 선정 이유

  • 쿠키런 모티브 — 3D 공간 상에서 점진적 가속 질주와 장애물 회피 결합.
  • 효율적 개발 — 학교 건물을 베이스로 구성해 공간 디자인 난이도를 타협.
1. 개요

인게임 뷰

로우폴리 3D 복도 에셋 배치.
스피드감 향상을 위해 캐릭터가 질주할 때 카메라 화각(FOV)을 동적으로 팽창시킵니다.

2. 설계

시스템 구조

모듈 결합도를 낮춰 각 시스템(플레이어 제어, NPC AI, 카메라 제어)이 충돌/세이브 이벤트를 주고받도록 구현했습니다.

2. 설계

주요 시스템

이동/회전

자동 가속, 마우스 감도 연동 좌우 조향, 점프·슬라이딩 연산.

데이터 관리

PlayerPrefs 활용 볼륨 및 감도 로컬 파일 세이브.

카메라 HUD

이동 속도 비례 앵글 조절 및 충돌 감지 연출.

2. 설계

로직 흐름

  • 입력 차단 — UI 열림/클리어/사망 시 컨트롤 값 수집 원천 배제.
  • 물리 주행 — 속도 = 기본속도 + 가속 + 대쉬 - 피격 감속 매 프레임 계산.
  • 충돌 분기 — 클리어존 도달 처리 및 장애물 충돌 라이프 감소 연동.
3. 설계 상세

상태 기반 타이머 관리

무거워지기 쉬운 FSM(상태머신) 클래스를 과감히 배제했습니다.
대신 hitCooldownTimer, slowTimer 등 float 변수를 매 프레임 업데이트에서 차감하여 연산 성능을 최적화했습니다.

3. 설계 상세

AABB 충돌 판정

자동으로 이동하는 플레이어와 불규칙 궤도로 횡단하는 NPC 간의 최속 감지를 위해 Unity BoxCollider(AABB 기반 바운딩 충돌)를 활용했습니다.

3. 설계 상세

확률 가중치 행동 선택

학생 NPC AI 의사결정에 확률 가중치를 적용했습니다.
일정 시간(retargetInterval)마다 난수를 뽑아 사전에 분배된 가중치(좌회전, 우회전, 정지, 직진)에 따라 목표지를 새롭게 지정합니다.

3. 설계 상세

선형 보간 (Lerp)

난수 선택 목표 지점으로 캐릭터가 워프하지 않도록 Vector3.Lerp와 MoveTowards를 믹스 연산하여, 미끄러지듯 이동하는 물리 주행을 구현했습니다.

4. 핵심 알고리즘

BFS 격자 탐색

NPC가 맵 외곽 벽에 막혀 가만히 멈춰 서 있는 오작동을 차단하기 위해 격자 맵 BFS(너비 우선 탐색)로 최단경로를 찾아 주행하도록 코딩했습니다.

4. 핵심 알고리즘

좌표계 실시간 변환

  • WorldToCell — 월드 실수형 Vector3를 그리드 정수형 Vector2Int 인덱스로 계산.
  • CellToWorld — BFS 탐색으로 찾아낸 최단경로 셀들을 다시 3D 월드 waypoint 실수 좌표로 역변환.
4. 핵심 알고리즘

탐색 다이어그램

인접 인덱스를 순차 방문하며, 벽으로 처리된 락커나 교실 구조물 인덱스 검사(bfsObstacles.Contains)를 거쳐 장애물을 능동적으로 비껴갑니다.

6. 코드 분석

Obstacle.cs (BFS)

private void BuildBfsPath(Vector3 destination) {
    path.Clear();
    pathIndex = 0;
    float cellSize = Mathf.Max(0.1f, bfsCellSize);
    Vector2Int start = WorldToCell(transform.position, cellSize);
    Vector2Int goal = WorldToCell(destination, cellSize);
    
    Queue<Vector2Int> queue = new Queue<Vector2Int>();
    Dictionary<Vector2Int, Vector2Int> previous = new Dictionary<Vector2Int, Vector2Int>();
    queue.Enqueue(start);
    previous[start] = start;

    while (queue.Count > 0) {
        Vector2Int current = queue.Dequeue();
        if (current == goal) break;
        foreach (Vector2Int dir in directions) {
            Vector2Int next = current + dir;
            if (previous.ContainsKey(next)) continue;
            previous[next] = current;
            queue.Enqueue(next);
        }
    }
}
최적화 설계
- sqrMagnitude 가시거리 연산을 적용하여 플레이어 근접 NPC만 깨어나 BFS를 수행하도록 최적화.
6. 코드 분석

Spawner.cs (배치 간격)

public void SpawnBatch() {
    GameObject[] targets = GameObject.FindGameObjectsWithTag(targetTag);
    int spawned = 0;
    while (spawned < spawnCount) {
        GameObject target = targets[Random.Range(0, targets.Length)];
        Vector3 pos = GetTopPosition(target) + Vector3.up * spawnHeightOffset;
        if (IsFarEnough(pos)) {
            Instantiate(obstaclePrefabs[Random.Range(0, obstaclePrefabs.Length)], pos, Quaternion.identity);
            usedPositions.Add(pos);
            spawned++;
        }
    }
}
스폰 겹침 방지
- IsFarEnough() 연산으로 이미 스폰된 객체 간의 거리를 검사하여 겹쳐서 생기는 렌더링 버그 차단.

7. 코드 분석

FollowCamera.cs (진동)

private Vector3 GetShakeOffset() {
    if (shakeTimer <= 0f) return Vector3.zero;
    
    shakeTimer -= Time.deltaTime;
    float power = shakeStrength * (shakeTimer / shakeDuration);
    
    float x = (Mathf.PerlinNoise(Time.time * shakeFrequency, 0.17f) - 0.5f) * 2f;
    float y = (Mathf.PerlinNoise(0.41f, Time.time * shakeFrequency) - 0.5f) * 2f;
    
    return transform.right * x * power + transform.up * y * power;
}
Perlin Noise 진동
- 랜덤 난수가 유발하는 멀미를 개선하기 위해 수학적으로 연속된 Perlin Noise 함수를 사용해 부드럽고 묵직한 진동 구현.
