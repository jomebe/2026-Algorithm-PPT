확률 가중치 기반 행동 선택

예측 불가능한 동적 장애물 구현의 핵심

학교 복도를 돌아다니는 학생 NPC의 고정되지 않은 무작위적 이동을 구현하기 위해 확률 가중치 알고리즘을 적용했습니다.

NPC 내부 타이머가 지정된 만료 시간(retargetInterval)에 도달할 때마다 난수를 생성합니다. 생성된 난수의 범위에 따라 좌회전, 우회전, 정지, 혹은 직진 가중치를 매겨 다음 타겟 목적지를 능동적으로 선택하도록 만들었습니다.

3. 적용 자료구조 및 알고리즘

선형 보간을 활용한 경로 평활화

순간이동 현상 방지 및 자연스러운 슬라이딩 이동

무작위 알고리즘으로 선정된 목적지 좌표로 NPC 캐릭터 오브젝트가 한순간에 텔레포트 이동한다면 게임성이 극도로 훼손됩니다.

목표 좌표 waypoint를 항해 매 프레임 독립적인 선형 보간인 Vector3.Lerp()와 MoveTowards() 물리 결합 방식을 혼용 적용하여, NPC가 미끄러지듯 자연스럽고 미려한 곡선을 그리며 이동하게 설계했습니다.
4. 핵심 알고리즘: BFS

BFS 격자 탐색 도입 이유

왜 BFS(너비 우선 탐색)를 선택했는가

장애물 NPC들이 빈 복도 구역을 그냥 직진하여 맵 밖으로 나가버리거나 물리 벽에 막혀 가만히 멈춰 서 있는 어색한 버그를 방지하고 싶었습니다.

따라서 맵 복도를 일정한 셀 그리드로 인식하고, 현재 위치에서부터 타겟 위치까지 최단 경로를 탐색해 가며 순차 이동하도록 BFS 알고리즘을 탐색 기초 뼈대로 채택했습니다. DFS에 비해 가로지르는 격자 형태 주행의 최단거리 최적화에 가장 알맞기 때문입니다.

4. 핵심 알고리즘: BFS

BFS 격자 좌표 변환 메커니즘

실수 좌표(World Space) 와 정수 좌표(Grid Cell)의 실시간 통신
  • WorldToCell(Vector3 worldPos)

    NPC가 서 있는 실제 유니티 월드 소수점 좌표에서 스폰 위치(Origin)를 차감한 뒤 격자 단위 크기(bfsCellSize)로 나누어 반올림(RoundToInt) 처리, 2차원 정수 좌표 Vector2Int(x, z)로 압축 변환합니다.

  • CellToWorld(Vector2Int cellPos)

    BFS 큐를 통과해 산출된 경로 노드 셀 위치에 셀 크기를 곱하고 스폰 위치 오프셋을 재가산하여, 3D 월드 상의 waypoint 실수형 좌표 벡터로 최종 변환 및 복원합니다.

4. 핵심 알고리즘: BFS

BFS 최단경로 탐색 모델

시작점으로부터 인접한 셀을 넓게 큐에 삽입하며 목표에 접근하는 BFS 알고리즘
벽(장애물) 충돌 자동 차단

BFS 루프를 수행하는 도중 탐색 큐에 삽입하기 직전, 장애물 벽이 설정된 정수 인덱스 검사(bfsObstacles.Contains())를 거쳐 해당 셀 방문을 원천 배제합니다. 이로써 자연스럽게 장애물을 감지하고 비껴가는 스마트 경로를 사전 설계하게 됩니다.

6. 핵심 소스코드 분석

Obstacle.cs (BFS & LERP)

// BFS 큐 연산 및 타겟 경로 추출 핵심 로직
private void BuildBfsPath(Vector3 destination) {
    path.Clear();
    pathIndex = 0;
    float cellSize = Mathf.Max(0.1f, bfsCellSize);
    Vector2Int start = WorldToCell(transform.position, cellSize);
    Vector2Int goal = WorldToCell(destination, cellSize);
    
    Queue<Vector2Int> queue = new Queue<Vector2Int>();
    Dictionary<Vector2Int, Vector2Int> previous = new Dictionary<Vector2Int, Vector2Int>();
    queue.Enqueue(start);
    previous[start] = start;

    while (queue.Count > 0) {
        Vector2Int current = queue.Dequeue();
        if (current == goal) break;
        foreach (Vector2Int dir in directions) {
            Vector2Int next = current + dir;
            if (previous.ContainsKey(next)) continue;
            previous[next] = current;
            queue.Enqueue(next);
        }
    }
}
큐(Queue)와 맵(Dictionary)의 협업:

- BFS의 탐색 순서를 보장하기 위해 선입선출 자료구조인 Queue<Vector2Int>를 활용했습니다.

- 방문 이력(Visited) 기록 및 최단 경로 역추적을 위한 부모 노드 연결 정보를 Dictionary<Vector2Int, Vector2Int>에 기록하여 최단 경로 복원을 O(N) 시간복잡도로 해결했습니다.

Spawner.cs (최소 스페이싱)

// NPC 스포너 배치 충돌 방지 로직
public void SpawnBatch() {
    GameObject[] targets = GameObject.FindGameObjectsWithTag(targetTag);
    int spawned = 0;
    while (spawned < spawnCount) {
        GameObject target = targets[Random.Range(0, targets.Length)];
        Vector3 pos = GetTopPosition(target) + Vector3.up * spawnHeightOffset;
        // 스폰 오브젝트 간 겹침 방지 검사
        if (IsFarEnough(pos)) {
            Instantiate(obstaclePrefabs[Random.Range(0, obstaclePrefabs.Length)], pos, Quaternion.identity);
            usedPositions.Add(pos);
            spawned++;
        }
    }
}
안정적 씬 오토스포닝:

- 씬 안의 임의 위치 타겟 오브젝트 리스트를 파악한 뒤, 스폰 높이 보정치를 얹은 후보 좌표를 구합니다.

- IsFarEnough() 연산으로 이미 배치된 NPC 후보군들과의 제곱거리(sqrMagnitude)를 조사해, 서로 너무 밀집되어 한 위치에 NPC가 스택되는 물리 렌더링 겹침 버그를 완전히 차단합니다.

PlayerController.cs (피격 및 제어)

// 피격 데미지 감소 및 디버프 상태 트리거
public void HitObstacle(string obstacleName) {
    if (cleared || hitCooldownTimer > 0f) return;
    
    hitCooldownTimer = hitCooldown; // 무적 돌입
    slowTimer = slowDuration;       // 일시 감속
    currentSpeed = Mathf.Max(startSpeed, currentSpeed - hitSpeedLoss);
    hearts = Mathf.Max(0, hearts - 1);
    heartUI?.SetHearts(hearts, maxHearts);
    
    followCamera?.Shake(0.45f, hearts <= 0 ? 1.2f : 0.75f);
    
    if (hearts <= 0) {
        Die(); // 게임오버 루틴
    }
}
피격 데미지 연속 계산 방지:

- 다단 히트 버그를 막기 위해 피격 즉시 hitCooldownTimer를 가동해 일정 프레임 동안 무적 판정을 부여합니다.

- 피격 시 플레이어가 인지할 수 있는 명확한 피드백을 전달하기 위해, 속도를 영구적으로 감속시키는 패널티와 동시에 카메라 진동 트리거를 송신합니다.
7. 핵심 소스코드 분석

FollowCamera.cs (Perlin Noise 셰이크)

// Perlin Noise를 응용한 부드럽고 타격감 넘치는 진동 오프셋
private Vector3 GetShakeOffset() {
    if (shakeTimer <= 0f) return Vector3.zero;
    
    shakeTimer -= Time.deltaTime;
    float power = shakeStrength * (shakeTimer / shakeDuration);
    
    float x = (Mathf.PerlinNoise(Time.time * shakeFrequency, 0.17f) - 0.5f) * 2f;
    float y = (Mathf.PerlinNoise(0.41f, Time.time * shakeFrequency) - 0.5f) * 2f;
    
    return transform.right * x * power + transform.up * y * power;
}
Perlin Noise 진동의 우수성:

- 단순 Random.insideUnitSphere를 사용할 경우 값이 거칠게 튀어 시각적인 불쾌감과 3D 어지러움증을 유발할 수 있습니다.

- 수학적으로 연속되고 자연스러운 난수 흐름인 Mathf.PerlinNoise를 활용해, 마치 손으로 카메라를 들고 찍는 듯한 생동감 있고 부드러운 흔들림 효과를 연출했습니다.
